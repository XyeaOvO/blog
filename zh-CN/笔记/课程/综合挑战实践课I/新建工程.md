新建项目看似简单，但对于Proteus来说，**选对第一步能省去后面很多麻烦**。

Proteus 8 引入了“工程向导”（New Project Wizard），这和旧版本直接画图不一样。作为学生，你通常会遇到两种情况：
1.  **纯电路模拟**（数电/模电作业）。
2.  **单片机仿真**（51/STM32/Arduino，需要写代码）。

推荐采用**“标准流程 + 外部编译”**的最佳实践，这是最稳定、最不容易报错的方法。

---

### 标准新建项目流程（一步步来）

打开 Proteus 8（记得右键管理员运行），点击首页左上角的 **"New Project"**。

#### 第一步：命名与路径（Name & Path）**【最重要的一步】**
*   **Name**: 给项目起个名。
    *   **警告**：**千万不要用中文！不要有空格！** 最好是全英文或“英文+数字”（例如 `Lab_01_LED`）。
*   **Path**: 选择保存位置。
    *   **建议**：建立一个专门的文件夹（如 `D:\Proteus_Work`），不要直接扔桌面。**路径中也尽量不要包含中文**，否则仿真时可能会报“File lookup failed”。
*   点击 **Next**。

#### 第二步：原理图模板（Schematic）
*   选择 **"Create a schematic from the selected template"**（从模板创建原理图）。
*   在列表中选择 **"DEFAULT"** 或 **"Landscape A4"**（横向A4纸）。
    *   *避坑：如果选了 "Do not create a schematic"，你就没有画图的地方了。*
*   点击 **Next**。

#### 第三步：PCB布局（PCB Layout）
*   **学生党通常选**：**"Do not create a PCB layout"**。
    *   *解释*：除非老师明确要求你画PCB板子去打样，否则我们只做仿真，不需要PCB界面，选了只会增加软件卡顿的概率。
*   点击 **Next**。

#### 第三步半：固件设置（Firmware）**【关键分歧点】**
这里会问你要不要创建固件（Create Firmware Project）。

*   **最佳实践建议**：**选择 "No Firmware Project"**。
*   *为什么？*
    *   Proteus自带的代码编辑器不好用。
    *   **主流做法**是：用 Keil (写51/STM32) 或 Arduino IDE 写代码，编译生成 `.hex` 文件，然后手动导入 Proteus。这样代码和电路分离，调试更灵活。
*   点击 **Next**，然后点击 **Finish**。

---

### 项目建立后的“黄金三步”

新建好项目后，不要急着画图，先做这三件事，效率翻倍。

#### 1. 加载单片机程序（如果你做单片机实验）
既然刚才选了 "No Firmware"，现在怎么把代码放进去？
1.  按 `P` 键，搜索并放好你的单片机（比如 `AT89C51` 或 `ARDUINO UNO`）。
2.  **双击** 单片机元件。
3.  在 **"Program File"** 这一栏，点击文件夹图标。
4.  找到你在 Keil 或 Arduino IDE 里编译好的 **`.hex`** 文件，打开它。
5.  点击 OK。
    *   *注：这样你就把“大脑”装进单片机了。*

#### 2. 设置电源网格（防止VCC报错）
有些隐藏引脚的元件（如74系列芯片）可能会报错说没电。
*   点击顶部菜单 **Design** -> **Configure Power Rails**。
*   把 `VCC` 和 `VDD` 添加到右侧的 `VCC` 网络中（通常默认是对的，但如果报错就来这里检查）。

#### 3. 开启对齐网格
*   确保你的视图里有“点状网格”。
*   如果没有，按快捷键 `G` 切换，或者在菜单 `View` -> `Toggle Grid`。
*   **必须开启网格**，否则你画的导线很可能看起来连上了，实际没连上（虚连接），排查起来能把人气死。

---

### 总结：学生党最稳妥的套路

1.  **New Project** -> 英文名、无中文路径。
2.  **Schematic** -> 选 Default。
3.  **PCB** -> 选 Do not create。
4.  **Firmware** -> 选 **No Firmware** (重点！)。
5.  **Finish** -> 双击单片机手动加载外部生成的 HEX 文件。

这套流程是很多学长“血泪教训”总结出来的，能避开Proteus内部编译器导致的各种莫名其妙的Bug。
